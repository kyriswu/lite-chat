# 系统提示词预算保护设计

## 目标

系统提示词必须进入每一次上游模型请求，不能因上下文 Token 预算被整段省略。

## 策略

系统提示词优先于历史消息占用上下文预算。若其估算长度达到或超过预算，保留开头内容并在末尾加入截断标记；随后不再选择历史消息。本次用户消息仍保留，保证请求可执行。

未超限时，系统提示词保持原文，历史消息仅使用预算余量。该裁剪只影响发送给上游模型的载荷，不会修改数据库中的原始提示词或聊天记录。

## 验证

验证短系统提示词原样保留、等于或超过预算的提示词保留开头且带截断标记，以及超长提示词仍作为 `system` 消息进入 Chat Completions 和 Responses 两种请求格式。
