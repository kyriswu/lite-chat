# 发送防重、供应商保存状态与默认供应商

## 目标

避免快速连续发送造成重复消息；让供应商保存操作有明确的进行中反馈；支持唯一的全局默认供应商。

## 设计

### 发送防重

聊天页面维护独立的提交中状态。请求从开始到完成（包括异常）期间，忽略新的发送触发。流式响应期间仍可使用既有的停止按钮中断响应，因此提交锁不改变停止语义。

### 保存状态

供应商表单提交时，保存按钮显示“保存中…”并禁用；请求结束后恢复原文案和可用状态。这样既提供反馈，也避免重复提交。

### 默认供应商

后台表单提供“设为默认供应商”开关，列表显示默认标识。服务端在创建或更新时处理 `isDefault`：设为默认时，在一个数据库事务内取消其余全局供应商的默认标记，再保存当前项；创建首个供应商时自动设为默认。取消当前默认的请求不会让系统失去默认项。

聊天端继续优先选择 `isDefault` 供应商，并保留首项兜底，以兼容历史数据。

## 验证

检查快速的回车和点击不会并发进入发送请求；检查保存期间按钮状态；检查创建、切换、编辑默认供应商后仅有一项默认，且聊天端取该项。
