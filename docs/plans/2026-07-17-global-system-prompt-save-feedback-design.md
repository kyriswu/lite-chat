# 全局系统提示词保存反馈设计

## 目标

用户在聊天页右上角的“全局提示”弹窗保存系统提示词后，获得明确的成功或失败反馈。

## 方案

在弹窗操作区中新增专用状态文本：

- 保存请求期间禁用“保存”按钮，防止重复提交。
- 请求成功后更新本地 `currentUser`，在弹窗内显示绿色“保存成功”，并在约两秒后自动清除。
- 请求失败时保留弹窗和输入内容，在同一位置显示错误信息；不关闭弹窗。
- 成功后弹窗保持打开，以便用户确认反馈或继续修改。

不修改既有 API、数据库结构和全局页面状态栏。

## 验证

手动确认成功保存显示提示、短暂禁用按钮，并且请求失败时能看到错误且输入内容未丢失。
