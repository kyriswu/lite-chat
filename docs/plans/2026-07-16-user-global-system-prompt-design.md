# 用户全局系统提示词设计

## 背景

当前系统提示词保存于 `conversations.system_prompt`，用户需要在每个对话中重复设置。用户确认将其改为按用户全局生效，并且不迁移任何已有会话的提示词。

## 方案

在 `users` 表增加可空的 `global_system_prompt` 字段。后端提供仅限当前登录用户读取和更新该字段的接口。前端的顶部“系统提示”入口改为读取、编辑这份全局配置，不再创建或更新对话来保存它。

发送消息时，后端从当前用户读取 `global_system_prompt`，并将其用作系统提示词。若选择技能，维持现有顺序：技能提示词在前，用户的全局提示词在后。历史对话的 `conversations.system_prompt` 保留在数据库中以兼容旧数据，但不会再读取或写入。

## 数据流

1. 登录后的客户端请求当前用户的全局系统提示词，并缓存到页面状态。
2. 用户通过顶部弹窗保存，客户端调用用户设置接口。
3. 后端更新当前用户的字段；下一次发送（包括任何历史对话）读取该字段并应用。

## 错误处理与验证

- 未登录时沿用现有访问控制。
- 空字符串标准化为 `NULL`，表示不注入全局系统提示词。
- 数据库迁移使用 `ADD COLUMN IF NOT EXISTS`，不会把历史会话值复制到用户字段。
- 验证新旧对话均从用户字段取值，且选择技能时仍会合并两类提示词。
