# 聊天上下文 Token 节省设计

## 目标

在不增加模型调用的前提下，优先降低每次聊天请求携带的上下文 Token 消耗。

## 策略

全局聊天设置增加三个可调项：上下文消息数、上下文 Token 预算、单条历史消息 Token 上限。默认值分别为 8 条、4,000 token 和 800 token。

上下文构建始终保留 System Prompt 与本次用户消息。历史记录从最新消息向前挑选，用户消息和助手回复按完整轮次保留；超出单条上限的历史内容只保留末尾并附带截断标识。随后再应用总 Token 预算。

## 边界与验证

历史消息的裁剪仅影响发送给模型的上下文，不修改数据库中的原始消息。图片消息保持原样。验证设置保存、边界值钳制，以及消息数、单条和总预算同时生效的上下文构建。
